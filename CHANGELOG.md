## [v2.4.5] — 2026-06-06

### Исправлено

- **Взятие просроченных заявок** — исполнитель может взять на себя нераспределённую заявку со статусом `overdue` (кнопка «Взять», массовое «Взять на себя», `assign_self` на сервере).

## [v2.1.0] — 2026-06-13

Патч-релиз после v2.0.0: исправления UX исполнителя, геолокации, CV и стабильности сервера.

### Добавлено

- **Кнопка «Завершить» в модалке «Открыть»** (`TaskModal`) — исполнитель может загрузить фото и завершить заявку без повторного входа.
- **Блок «Данные при закрытии»** в карточке выполненной заявки: устройство, ОС/платформа, геолокация (ссылка на карту), отчёт.
- **Запрос геолокации при входе** — модуль `client/src/utils/geolocation.js`; диалог браузера показывается при клике «Войти» (в цепочке user gesture); координаты кэшируются в `localStorage`.
- **Сворачиваемые фильтры на мобильных** — на экранах &lt; 768px блок фильтров заявок по умолчанию скрыт, раскрывается кнопкой «Фильтры» с бейджем активных фильтров.
- **Завершение просроченных заявок** — кнопка «Завершить» доступна для статусов `in_progress`, `overdue`, `returned`, `emergency` (хелпер `canExecutorCompleteTask`).
- **Колонки в Excel-экспорте** — устройство закрытия, ОС, широта, долгота.

### Исправлено

- **CV зависал на «Проверка CV...»** — синхронная проверка при загрузке фото, очередь `runInCvQueue`, таймаут 120 с для upload/complete, предзагрузка модели при старте сервера.
- **`ReferenceError: cachePhotos`** в `PhotoUpload.jsx` после успешной загрузки фото.
- **Синтаксическая ошибка в `server/routes/tasks.js`** (`??` и `||` без скобок) — сервер не стартовал, nginx отдавал **502**.
- **Сравнение `assigned_to`** — через `Number()`, чтобы кнопка «Завершить» не пропадала из‑за разных типов id.
- **Геолокация при закрытии** — запрос в момент «Завершить» часто не показывал диалог (истёк user gesture); теперь используется кэш + тихое обновление при уже выданном разрешении.

### Технические детали

| Область | Файлы | Поведение |
|---------|-------|-----------|
| Геолокация | `client/src/utils/geolocation.js`, `Login.jsx`, `AuthContext.jsx`, `client/src/utils.js` | См. таблицу импортов ниже |
| Закрытие заявки | `Tasks.jsx`, `server/routes/tasks.js` | `closed_device`, `closed_os`, `closed_latitude`, `closed_longitude` всегда пишутся при `status=completed`; fallback User-Agent с сервера |

#### Импорты геолокации (важно для сборки)

| Файл | Правильный импорт |
|------|-------------------|
| `client/src/utils.js` | `from './utils/geolocation.js'` |
| `client/src/pages/Login.jsx` | `from '../utils/geolocation'` |
| `client/src/context/AuthContext.jsx` | `from '../utils/geolocation'` |

> `utils.js` находится в `src/`, а `geolocation.js` — в подпапке `src/utils/`. Путь `./geolocation.js` **неверен** и ломает `vite build`.

#### localStorage (геолокация)

| Ключ | Содержимое |
|------|------------|
| `geo_position_cache` | `{ latitude, longitude, at }` |
| `geo_permission_state` | `granted` / `denied` (fallback, если Permissions API недоступен) |

#### Сервер: поля при закрытии заявки

При `PATCH /api/tasks/:id` с `status: completed` всегда записываются `closed_device`, `closed_os`, `closed_latitude`, `closed_longitude`. Выражение в коде: `closed_device ?? (ua || null)` — **нельзя** писать `??` и `||` без скобок (SyntaxError, 502).

### Деплой

```bash
cd ~/control-app && git pull && bash deploy/build-client.sh && pm2 restart control-app
```

### Проверка после деплоя

1. Вход с мобильного → запрос геолокации при «Войти».
2. Заявки → фильтры свёрнуты, раскрываются по кнопке.
3. Просроченная заявка исполнителя → видна кнопка «Завершить».
4. Завершение с фото → в карточке блок «Данные при закрытии».
5. `pm2 logs control-app` → «Сервер запущен», при CV — «CV-модель готова».

> **Геолокация** работает только по **HTTPS** (или `localhost`). На HTTP браузер не покажет диалог.
